# ModoFit E-commerce System

## Overview
ModoFit is a robust e-commerce platform designed for selling health and fitness products. It features a modern Vue.js frontend and a scalable NestJS backend, integrated with Cloudinary for image management and Mercado Pago for payments.

## Architecture
The system follows a microservices-inspired modular architecture within a monorepo structure:

- **Frontend**: Vue 3 + Vite + Tailwind CSS
    - State Management: Pinia
    - Routing: Vue Router
    - HTTP Client: Axios
- **Backend**: NestJS (Node.js)
    - Database: PostgreSQL with TypeORM
    - Authentication: JWT (JSON Web Tokens)
    - API Documentation: Swagger (OpenAPI)
    - Logging: Winston (Daily rotation)
    - File Storage: Cloudinary

## Prerequisites
- Node.js (v18+)
- PostgreSQL
- Cloudinary Account
- Mercado Pago Account

## Setup Instructions

1.  **Clone the repository**
    ```bash
    git clone <repository-url>
    cd modofit
    ```

2.  **Install Dependencies**
    ```bash
    npm install
    cd frontend
    npm install
    cd ..
    ```

3.  **Environment Configuration**
    Create a `.env` file in the root directory with the following variables:
    ```env
    # Database
    DB_HOST=localhost
    DB_PORT=5432
    DB_USERNAME=postgres
    DB_PASSWORD=your_password
    DB_DATABASE=modofit

    # JWT Authentication
    JWT_SECRET=your_jwt_secret

    # Cloudinary (Image Storage)
    CLOUDINARY_CLOUD_NAME=your_cloud_name
    CLOUDINARY_API_KEY=your_api_key
    CLOUDINARY_API_SECRET=your_api_secret

    # Mercado Pago
    MERCADOPAGO_ACCESS_TOKEN=your_access_token

    # Application
    PORT=3000
    NODE_ENV=development
    ```

4.  **Run the Application**
    - **Backend**:
        ```bash
        npm run start:dev
        ```
    - **Frontend**:
        ```bash
        cd frontend
        npm run serve
        ```

## API Documentation
The backend includes interactive API documentation generated by Swagger.
Once the server is running, access it at:
**http://localhost:3000/api/docs**

## Testing
Run unit tests for the backend:
```bash
npm run test
```

## Deployment (Vercel)
This project is configured for deployment on Vercel.
- The `logs/` directory is ephemeral and not persisted in Vercel. Logs are captured via `stdout` and viewable in the Vercel Dashboard.
- Ensure all environment variables are correctly set in the Vercel Project Settings.

## Project Structure
- `src/`: Backend source code
    - `products/`: Product management module
    - `sales/`: Sales and statistics module
    - `users/`: User management and authentication
    - `storage/`: Cloudinary integration
    - `payments/`: Mercado Pago integration
- `frontend/src/`: Frontend source code
    - `components/`: Vue components
    - `views/`: Page views
    - `stores/`: Pinia state stores
